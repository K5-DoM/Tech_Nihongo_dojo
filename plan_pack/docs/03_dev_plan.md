# 03. 開発計画書（12週間）

## 全体方針
- **先に価値検証、後で拡張**
- 「音声」「凝ったUI」より「評価品質」「継続率(UX)」を優先

> **JTC就活向け・4週間集中**: 大企業技術職のポートフォリオで「設計→実装→評価→運用」を最短で示したい場合は、優先順位・ロードマップ・DoD・リスクを **15_jtc_portfolio_dev_policy.md** に集約している。そちらを優先し、課金・成果物は Could に回す。

---

## Phase 1（Week 1-4）: コアMVP

### Week 1
- リポジトリ初期化（monorepo）
- Cloudflare Workers + Hono雛形
- Supabase接続/Auth導入

- **面接で話せる論点**: なぜ monorepo と Cloudflare Workers を選んだか（スキーマ共有・エッジのレイテンシ・個人開発のコスト）
- **確認質問（私が答えるべき）**: 「認証を Supabase にした理由と、他案と比較したトレードオフは？」

### Week 2
- `/api/chat` 実装（Structured Outputs）
- 5軸評価JSONスキーマ定義
- 最小チャットUI実装

- **面接で話せる論点**: LLM 出力をどう契約化したか（Zod + Structured Output でパース失敗をどう扱うか）
- **確認質問（私が答えるべき）**: 「JSON が返ってこないときのリトライ・フォールバックをどう設計したか？」

### Week 3
- 会話履歴・評価保存
- 弱点タグ抽出と保存
- セッション一覧画面

- **面接で話せる論点**: セッション終了時の 5 軸評価生成（POST /api/interviews/:id/finish）と、弱点履歴を次回プロンプトにどう渡すか
- **確認質問（私が答えるべき）**: 「弱点タグを次回面接の質問に反映する設計を、どう実装で担保しているか？」

### Week 4
- αテスト（5〜10人）
- バグ修正
- 指摘根拠表示・再生成UI

- **面接で話せる論点**: Gate A（セッション完了率・評価JSON失敗率）を指標にした検証と、αで得たフィードバックの優先順位の付け方
- **確認質問（私が答えるべき）**: 「α で一番効いた改善と、見送った要望をどう判断したか？」

**Gate A（通過条件）**
- セッション完了率 >= 70%
- 評価JSON失敗率 <= 3%

## Phase 2（Week 5-8）: 収益化

### Week 5
- 成果物生成API（自己紹介/研究説明/深掘りQA）
- Markdown出力 + コピーUI

- **面接で話せる論点**: 会話ログから成果物（30秒自己紹介・60秒研究説明・深掘りQA）を生成する入力設計と、プロンプトの役割分担
- **確認質問（私が答えるべき）**: 「成果物の品質をユーザーにどう伝え、フィードバックをどう拾うか？」

### Week 6
- Stripe Checkout
- Webhookでticket加算
- 購入履歴画面

- **面接で話せる論点**: Stripe Webhook の冪等性・署名検証と、チケット加算の監査（billing_events）の取り方
- **確認質問（私が答えるべき）**: 「Webhook が重複して届いた場合、二重付与を防ぐ仕組みは？」

### Week 7
- 利用規約・プライバシーポリシー
- 免責モーダル実装
- 問い合わせ導線

- **面接で話せる論点**: 課金サービスとして免責・個人情報の取り扱いをどこまで明文化したか
- **確認質問（私が答えるべき）**: 「AI 評価の誤りについて、利用規約でどう免責しつつユーザーに伝えているか？」

### Week 8
- クローズドβ（20人規模）
- 課金動線改善

- **面接で話せる論点**: Gate B（無料→有料転換率・7日継続率）を見た上での課金動線の変更判断
- **確認質問（私が答えるべき）**: 「β で転換率・継続率をどう計測し、何を変えたか？」

**Gate B（通過条件）**
- 無料→有料転換率 >= 10%
- 7日継続率 >= 25%

## Phase 3（Week 9-12）: 運用最適化

### Week 9
- KPIダッシュボード（簡易）
- API原価の可視化

- **面接で話せる論点**: 個人開発で KPI と API 原価をどう可視化し、スコープやプロンプト設計に反映するか
- **確認質問（私が答えるべき）**: 「原価が予想を超えた場合、どの機能・パラメータから見直すか？」

### Week 10
- プロンプトABテスト
- 弱点再提示ロジック改善

- **面接で話せる論点**: 評価品質・継続率を上げるためのプロンプト改善の仮説と、簡易 AB テストのやり方
- **確認質問（私が答えるべき）**: 「弱点再提示の有無で、どの指標を比較して効果を判断するか？」

### Week 11
- ドキュメント整備（README/設計図）
- 就活向けデモ台本作成

- **面接で話せる論点**: ポートフォリオとして「何をどこまで説明できる状態にしたか」（設計図・意思決定ログ・デモの流れ）
- **確認質問（私が答えるべき）**: 「面接官に 3 分でこのプロダクトの価値と自分の役割をどう伝えるか？」

### Week 12
- 公開ポスト（Qiita/Zenn）
- リリースノート v1.0

- **面接で話せる論点**: リリース後の振り返り（Gate C との比較・有料ユーザー数・月次黒字の見通し）
- **確認質問（私が答えるべき）**: 「v1.0 でやり切ったことと、意図的にスコープから外したことをどう説明するか？」

**Gate C（目標）**
- 有料ユーザー 5〜15
- 月次黒字化の見通し
