# 00_0. プロジェクト意図（Project Intent）

## 開発目的（就活で語るべき価値）

1. **成果に直結する学習環境**  
   理系留学生が「技術説明を日本語で構造化して伝える」「面接文脈で敬語を維持する」という課題に、反復訓練で対応できるSaaSを提供する。

2. **説明可能な開発経験の蓄積**  
   要件定義・設計・実装・運用まで一気通貫で手がけ、**なぜその技術選定をしたか・どのトレードオフを許容したか**を面接で説明できる成果物を残す。

3. **ポートフォリオとしての実運用レベル**  
   単なるデモではなく、認証・課金・評価・履歴を備えた「本番に近い」プロダクトとして構築し、スコープ管理と優先順位の判断経験を示す。

---

## 対象ユーザー・課題・成功指標

| 項目 | 内容 |
|------|------|
| **Primary ユーザー** | 日本就職を目指す理系留学生（大学生・大学院生、主に情報系）、日本語 N2〜N1 |
| **Secondary** | 日本語で技術説明を鍛えたい日本人学生 |
| **ペイン** | 研究説明が冗長/抽象的・敬語と技術説明の両立が難しい・練習相手・記録が残らない |
| **バリュー** | 理系面接特化・5軸ルーブリックで一貫評価・弱点履歴の次回反映・成果物（自己紹介/研究説明/深掘りQA）生成 |

**成功指標（3か月）**

- MVP公開（テキスト面接 + 5軸評価 + 履歴）
- 初期ユーザー 20〜50名、有料転換 5〜15名
- 面接練習 7日継続率 25%以上
- 「汎用AIではなく本サービスを継続利用する理由」がユーザーに伝わること

---

## 非目標（やらないこと）

- **MVPスコープ外**  
  - 完全リアルタイム音声面接（βで後続検討）  
  - 多言語UI（初期は日本語UIのみ）  
  - 企業向け管理ダッシュボード（B2B機能）

- **プロダクト方針**  
  - 採用結果の保証はしない  
  - 機密情報入力を禁止  
  - AI評価は補助であり、最終判断はユーザーが行う

- **開発方針**  
  - ゼロからの作り直しはしない（既存成果を活かして軌道修正）  
  - 個人開発の制約（1日2〜4時間）を超えたスコープ creep は受けない
